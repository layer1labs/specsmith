# Scenario: Requirement Drift Detection
## Problem statement
Detect and correct divergence between implemented behavior and documented requirements.

## Workflow steps
1. Introduce controlled requirement drift.
2. Run governance checks.
3. Identify mismatches.
4. Re-align implementation or requirements.

## Metrics to capture
- drift detection latency
- traceability precision
- audit completeness
- recovery quality after correction

## Comparison axes
- no governance
- Spec Kit
- OpenSpec
- BMAD
- direct agent
- specsmith governed
