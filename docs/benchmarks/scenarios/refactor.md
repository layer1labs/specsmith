# Scenario: Refactor
## Problem statement
Refactor an existing subsystem for maintainability without changing user-facing behavior.

## Workflow steps
1. Define invariant behavior expectations.
2. Refactor modules.
3. Run regression tests.
4. Evaluate governance outputs.

## Metrics to capture
- time to completion
- invariant preservation traceability
- audit completeness
- recovery quality

## Comparison axes
- no governance
- Spec Kit
- OpenSpec
- BMAD
- direct agent
- specsmith governed
