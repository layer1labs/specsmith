# 1.0 Release Criteria

This document defines measurable readiness criteria for SpecSmith 1.0.

## Core product stability criteria

- Stable CLI core published and frozen in the 1.0 stability contract (`docs/stability.md`).
- Stable generated file schemas for requirements/tests/work items/trace artifacts.
- Stable MCP governance tool schemas for public tool set.
- Migration test suite validating schema compatibility transitions (tracked in #218).

## Quality and verification criteria

- Type coverage target: at least 90% for typed public Python modules.
- Test coverage target: greater than 80% across core packages.
- CI quality gates green on supported platforms.
- No unresolved critical security findings at release cut.

## Security and governance criteria

- Security review completed against `docs/security-threat-model.md`.
- Responsible disclosure process active and documented in `SECURITY.md`.
- Audit chain verification included in release checklist.

## Documentation and adoption criteria

- Docs completeness for core workflows (init, preflight, verify, audit, release).
- Public 10-minute tutorial available (`docs/tutorials/10-minute-governed-change.md`).
- Plain-English glossary available for onboarding (`docs/concepts/plain-english-glossary.md`).
- OSS/commercial editions matrix published (`docs/editions.md`).
- Stability contract published (`docs/stability.md`).

## Upgrade path criteria

- 0.x to 1.0 upgrade path documented with migration notes.
- Breaking changes documented under dedicated changelog section.
- Automated migration checks and schema version assertions tracked in #218.
