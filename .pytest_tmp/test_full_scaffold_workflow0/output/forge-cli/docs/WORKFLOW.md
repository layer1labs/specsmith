# forge-cli — Development Workflow

## Work Loop

All development follows the six-step loop defined in AGENTS.md governance docs:
1. State loading → 2. Proposal → 3. Constraint check → 4. Bounded execution → 5. Verification → 6. Ledger update

## Milestones

1. **Scaffold** — governance files, directory structure, bootstrap scripts (current)
2. **Architecture** — concrete interface specifications, technology decisions
3. **Requirements** — formal requirements derived from architecture
4. **Test spec** — test cases linked to requirements
5. **Implementation** — primary code scaffold, first feature
6. **Verification** — all smoke tests passing, cross-platform validation
